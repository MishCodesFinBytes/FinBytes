Here’s a list of commonly used decorators in Python, organized by category:

1. Built-in Function Decorators

These are standard decorators provided by Python to modify the behavior of methods and functions.

@staticmethod
Marks a method as a static method, not bound to a class instance. 

class MyClass: 

  @staticmethod 
  def static_method(): 
    return "Static method"

@classmethod
Marks a method as a class method, which receives the class (cls) as its first argument. 

class MyClass: 
  
  @classmethod 
  def class_method(cls): 
    return f"Called on {cls.__name__}"

@property
Used to define getter methods that behave like attributes. 

class MyClass: 
  
  def __init__(self, value): 
    self._value = value 
    
  @property 
  def value(self): 
    return self._value

@functools.lru_cache
Caches results of a function to optimize performance. 

from functools import lru_cache 
  
@lru_cache 
def expensive_function(x): 
  return x * x

@functools.wraps
Preserves the original function’s metadata when writing custom decorators. 

from functools import wraps

def my_decorator(func): 
  @wraps(func) 
  def wrapper(*args, **kwargs): 
    print("Before function call") 
    return func(*args, **kwargs) 
return wrapper

2. Class-Related Decorators

@dataclass
Simplifies the creation of classes by auto-generating methods like __init__ and __repr__. 

from dataclasses import dataclass 

@dataclass 
class MyClass: 
  name: str 
  age: int

@abc.abstractmethod
Used in abstract base classes to declare methods that must be implemented in derived classes. 

from abc import ABC, abstractmethod 

class MyAbstractClass(ABC): 

  @abstractmethod 
  def abstract_method(self): 
    pass

3. Debugging and Testing Decorators

@unittest.skip
Skips a specific test in the unittest framework. 

import unittest 

class MyTest(unittest.TestCase): 

  @unittest.skip("Skip this test") 
  def test_example(self): 
    pass

@unittest.expectedFailure
Marks a test as expected to fail. 

@unittest.expectedFailure 
def test_fail(self): 
  self.assertEqual(1, 2)

4. Function-Execution Control Decorators

@contextlib.contextmanager
Simplifies writing context managers. from contextlib import contextmanager 

@contextmanager 
def my_context(): 
  print("Enter") 
    yield print("Exit")

@asyncio.coroutine
Marks a function as a coroutine in older versions of Python (now replaced by async def). 

import asyncio 

@asyncio.coroutine 
def my_coroutine(): 
  yield from asyncio.sleep(1)

5. Miscellaneous Decorators

@staticmethod and @classmethod
Used to define static and class methods in classes. (Already explained above.)

@functools.partial
Partially applies arguments to a function (technically not a decorator but frequently used like one).

6. Common Third-Party Decorators

@pytest.mark.parametrize
In the pytest framework, runs a test with multiple sets of parameters. 

import pytest 

@pytest.mark.parametrize("input,expected", [(2, 4), (3, 9)]) 
def test_square(input, expected): 
  assert input ** 2 == expected

@flask.route
Used in Flask to define HTTP routes. 

from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Hello, World!"

@click.command
Used in click for creating command-line interfaces. 

import click 

@click.command() 
def clicked(): 
  click.echo("Hello, I got CLICKED!")

7. Custom Decorators

Custom decorators can be created for various purposes.

def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("Before function call")
        result = func(*args, **kwargs)
        print("After function call")
        return result
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")
