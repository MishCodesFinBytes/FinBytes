In computer science, Big O Notation is a way to describe the efficiency of an algorithm. It allows us to estimate the worst-case performance of an algorithm in terms of time (how fast it runs) and space (how much memory it uses). Understanding Big O is crucial for writing efficient code, especially in data-intensive domains like fintech.

This post will explain the common types of complexities with Python code examples and a visual graph to illustrate the differences.

Common Big O Notation Levels

O(1) - Constant Time The algorithm's performance does not change with the size of the input. 

def get_first_element(lst): 
  return lst[0] # Accessing an element by index is O(1)

O(log n) - Logarithmic Time The algorithm reduces the problem size in each step. 

import math

def binary_search(arr, target):
    low, high = 0, len(arr) - 1
    iterations = 0
    while low <= high:
        iterations += 1
        mid = (low + high) // 2
        if arr[mid] == target:
            print(f"Iterations: {iterations}")
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1
    print(f"Iterations: {iterations}")
    return -1  # Target not found

O(n) - Linear Time The algorithm's performance grows linearly with input size. 

def find_max(lst):
    max_val = lst[0]
    iterations = 0
    for num in lst:
        iterations += 1
        if num > max_val:
            max_val = num
    print(f"Iterations: {iterations}")
    return max_val

O(n log n) - Quasilinear Time Common in efficient sorting algorithms like mergesort. 

def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    return merge(left, right)

def merge(left, right):
    sorted_arr = []
    iterations = 0
    while left and right:
        iterations += 1
        if left[0] < right[0]:
            sorted_arr.append(left.pop(0))
        else:
            sorted_arr.append(right.pop(0))
    sorted_arr.extend(left or right)
    print(f"Iterations in merge: {iterations}")
    return sorted_arr

O(n^2) - Quadratic Time Performance grows quadratically with the size of the input, often seen in nested loops. 

def bubble_sort(arr):
    n = len(arr)
    iterations = 0
    for i in range(n):
        for j in range(0, n-i-1):
            iterations += 1
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    print(f"Iterations: {iterations}")

O(2^n) - Exponential Time Performance doubles with each additional input, common in recursive algorithms without memoization. 

from itertools import permutations

def traveling_salesman(cities):

O(n!) - Factorial Time Seen in brute-force algorithms for solving problems like the traveling salesman problem.

Visualizing Big O Complexity

Below is a Python snippet to generate a visual graph comparing the growth of different complexities:

import matplotlib.pyplot as plt
import numpy as np

n = np.linspace(1, 10, 100)

plt.plot(n, np.ones_like(n), label="O(1)")
plt.plot(n, np.log2(n), label="O(log n)")
plt.plot(n, n, label="O(n)")
plt.plot(n, n * np.log2(n), label="O(n log n)")
plt.plot(n, n**2, label="O(n^2)")
plt.plot(n, 2**n, label="O(2^n)")

plt.ylim(0, 100)
plt.xlabel("Input size (n)")
plt.ylabel("Operations")
plt.title("Big O Complexity Growth")
plt.legend()
plt.grid(True)
plt.show()

The resulting graph helps visualize how different complexities grow as the input size increases. While O(1) and O(log n) scale well, complexities like O(n^2) and O(2^n) quickly become impractical.

  • Big O Notation is vital for assessing algorithm efficiency.

  • Understanding the growth rates helps in selecting the right algorithm for a problem.

  • Optimize algorithms for scalability by minimizing time and space complexity.

Solving the Same Problem with Different Complexities

To demonstrate the impact of different complexities, we will solve the same problem—finding the sum of the first n integers (for lack of a better idea, suggestions are welcome! )—using multiple approaches. Each approach will have a different complexity, and we will count the number of iterations for each.

Problem: Find the Sum of the First n Integers

We will solve this problem using four approaches:

  1. O(1) - Constant time: Using a mathematical formula.

  2. O(n) - Linear time: Iterating through a range.

  3. O(n log n) - Quasilinear time: Recursively divide and conquer.

  4. O(n^2) - Quadratic time: Nested loops.

Each solution will count the iterations and output the total.

import matplotlib.pyplot as plt

# O(1) Solution
def sum_o1(n):
    """Calculate the sum using a mathematical formula (constant time)."""
    iterations = 1  # Only one operation
    result = n * (n + 1) // 2
    print(f"O(1) iterations: {iterations}")
    return result, iterations

# O(n) Solution
def sum_on(n):
    """Calculate the sum by iterating through numbers 1 to n."""
    iterations = 0
    total = 0
    for i in range(1, n + 1):
        iterations += 1
        total += i
    print(f"O(n) iterations: {iterations}")
    return total, iterations

# O(n log n) Solution
def sum_onlogn_helper(arr, iterations):
    """Helper function to sum an array using divide and conquer."""
    if len(arr) <= 1:
        return sum(arr), iterations
    mid = len(arr) // 2
    iterations += 1
    left_sum, iterations = sum_onlogn_helper(arr[:mid], iterations)
    right_sum, iterations = sum_onlogn_helper(arr[mid:], iterations)
    return left_sum + right_sum, iterations

def sum_onlogn(n):
    """Calculate the sum using a divide-and-conquer approach."""
    arr = list(range(1, n + 1))
    total, iterations = sum_onlogn_helper(arr, 0)
    print(f"O(n log n) iterations: {iterations}")
    return total, iterations

# O(n^2) Solution
def sum_on2(n):
    """Calculate the sum using nested loops."""
    iterations = 0
    total = 0
    for i in range(1, n + 1):
        for j in range(1, i + 1):
            iterations += 1
            total += 1
    print(f"O(n^2) iterations: {iterations}")
    return total, iterations

# Compare the methods
n_values = [10, 50, 100, 200, 500]
o1_iterations = []
on_iterations = []
onlogn_iterations = []
on2_iterations = []

for n in n_values:
    _, iter_o1 = sum_o1(n)
    _, iter_on = sum_on(n)
