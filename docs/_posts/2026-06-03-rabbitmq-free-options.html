You want to experiment with RabbitMQ. What are your options without paying anything?

Option 1 — CloudAMQP free tier (Little Lemur)

The easiest starting point. Sign up at cloudamqp.com, create an instance on the free "Little Lemur" plan.

Free tier limits: 1 concurrent connection, 1 million messages/month, max 20 queues. Enough for learning and small projects.

You get a connection URL in the format: amqps://user:pass@hostname/vhost

No credit card required for the free plan.

Option 2 — Local Docker (recommended for development)

No limits, no internet required, management UI included.

# Start RabbitMQ with the management plugin enabled
docker run -d --name rabbitmq \
  -p 5672:5672 \
  -p 15672:15672 \
  rabbitmq:3-management

# Management UI: http://localhost:15672
# Default credentials: guest / guest

The management UI at port 15672 lets you see queues, exchanges, bindings, message rates, and publish/consume test messages manually — invaluable for learning.

Which should you use?

For learning RabbitMQ specifically: local Docker. Full control, management UI, no rate limits, reset whenever you want.

For a shared dev/staging environment: CloudAMQP free tier. Accessible from anywhere, zero ops.

For a Python project that just needs a task queue: consider Celery + Redis — you may already have Redis, and Celery handles the worker management, retry, and scheduling on top of it.
