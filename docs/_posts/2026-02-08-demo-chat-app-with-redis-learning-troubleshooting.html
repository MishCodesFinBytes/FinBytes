This is a learning/demo chat app that demonstrates Redis caching, issues, and troubleshooting, and can be run with Gunicorn (or Windows-friendly alternatives). I’ll make it step-by-step with prerequisites, including notes on cost.

Goals:

  1. Build a simple chat app using Redis + Flask + Pub/Sub.

  2. Demonstrate caching issues (misses, stale data, TTL problems).

  3. Show how to troubleshoot issues at different stages (app, server, Redis).

  4. Run with Gunicorn (Linux preferred) or Waitress (Windows-friendly).

Step 0: Prerequisites

💡 Tip: If you want Gunicorn on Windows, the easiest way is to run a Linux VM, WSL, or Docker container. Otherwise, use Waitress for Windows demo.

Step 1: Single-File Demo App – chat_cache_demo.py

import redis
import threading
import time
from flask import Flask, request, jsonify

# ------------------------------
# Redis Connection
# ------------------------------
r = redis.Redis(host='localhost', port=6379, db=0, decode_responses=True)

# ------------------------------
# Chat and Caching
# ------------------------------
class ChatCacheDemo:
    """Chat app with cache demo features"""

    MESSAGE_LIST_KEY = "chat_messages"  # Store last N messages

    @staticmethod
    def start_listener():
        pubsub = r.pubsub()
        pubsub.subscribe("global_chat")
        print("Chat listener started. Waiting for messages...")
        for message in pubsub.listen():
            if message['type'] == 'message':
                print(f"[CHAT LIVE] {message['data']}")

    @staticmethod
    def send_message(message):
        # Publish to live subscribers
        r.publish("global_chat", message)
        # Add to cache for history (TTL = 300s)
        r.lpush(ChatCacheDemo.MESSAGE_LIST_KEY, message)
        r.ltrim(ChatCacheDemo.MESSAGE_LIST_KEY, 0, 9)  # Keep last 10 messages
        r.expire(ChatCacheDemo.MESSAGE_LIST_KEY, 300)  # Messages expire after 5 minutes

    @staticmethod
    def get_cached_messages():
        # Demonstrate cache retrieval
        messages = r.lrange(ChatCacheDemo.MESSAGE_LIST_KEY, 0, -1)
        if not messages:
            print("Cache miss! Messages not found (expired or not populated).")
        return messages

# ------------------------------
# Start listener thread
# ------------------------------
threading.Thread(target=ChatCacheDemo.start_listener, daemon=True).start()

# ------------------------------
# Flask App
# ------------------------------
app = Flask(__name__)

@app.route("/chat", methods=["POST"])
def send_chat():
    """Send a chat message"""
    data = request.json
    message = data.get("message")
    if message:
        ChatCacheDemo.send_message(message)
        return jsonify({"status": "sent"})
    return jsonify({"status": "failed", "error": "No message provided"}), 400

@app.route("/history")
def chat_history():
    """Retrieve cached messages"""
    messages = ChatCacheDemo.get_cached_messages()
    return jsonify({"messages": messages})

@app.route("/ping")
def ping():
    return "Server running!"

# ------------------------------
# Run App
# ------------------------------
if __name__ == "__main__":
    print("Starting Chat Cache Demo App...")
    app.run(debug=True)

Step 2: How to Run Locally (Windows)

  1. Start Redis

    • Run redis-server.exe in your Redis folder (e.g., C:\Redis)

  2. Run Flask app

python chat_cache_demo.py

  • Console shows incoming live messages and cache debug info.

  1. Test API endpoints

  • Send message:

POST http://127.0.0.1:5000/chat
Content-Type: application/json
Body: {"message": "Hello demo chat!"}

  • Get cached messages:

GET http://127.0.0.1:5000/history

  • Ping server:

GET http://127.0.0.1:5000/ping

Step 4: Optional – Run with Waitress (Windows-friendly production server)

from waitress import serve
serve(app, host="0.0.0.0", port=5000)

  • Now app can handle multiple concurrent requests safely (like Gunicorn).

Step 5: Optional – Gunicorn (Linux / WSL / Docker)

# On Linux / WSL / Docker
pip install gunicorn
gunicorn -w 4 chat_cache_demo:app

  • -w 4 → 4 worker processes

  • You can demonstrate race conditions / caching issues with multiple Gunicorn workers
