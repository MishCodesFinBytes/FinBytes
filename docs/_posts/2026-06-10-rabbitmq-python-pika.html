pika is the standard Python client for RabbitMQ. It's low-level — you manage connections, channels, declarations, and acknowledgements yourself. That explicitness is useful for understanding what's happening.

Install

pip install pika

Minimal producer

import pika

connection = pika.BlockingConnection(
    pika.URLParameters('amqp://guest:guest@localhost/')
)
channel = connection.channel()

# Declare queue (idempotent — safe to call every time)
channel.queue_declare(queue='tasks', durable=True)

channel.basic_publish(
    exchange='',
    routing_key='tasks',
    body='hello world',
    properties=pika.BasicProperties(delivery_mode=2)  # persistent
)

print("Sent")
connection.close()

Minimal consumer

import pika

def callback(ch, method, properties, body):
    print(f"Received: {body.decode()}")
    ch.basic_ack(delivery_tag=method.delivery_tag)  # acknowledge

connection = pika.BlockingConnection(
    pika.URLParameters('amqp://guest:guest@localhost/')
)
channel = connection.channel()

channel.queue_declare(queue='tasks', durable=True)
channel.basic_qos(prefetch_count=1)  # one at a time
channel.basic_consume(queue='tasks', on_message_callback=callback)

print("Waiting for messages...")
channel.start_consuming()  # blocks

Important settings explained

durable=True — the queue survives a RabbitMQ restart. Without this, queues disappear on restart and messages are lost.

delivery_mode=2 — marks messages as persistent. Combined with durable=True, messages survive restarts.

basic_ack — tells RabbitMQ the message was processed. Without an ack, RabbitMQ redelivers the message when the consumer disconnects. Never forget to ack (or nack on failure).

prefetch_count=1 — don't send more than 1 unacked message to this consumer. Prevents one slow consumer from hoarding messages.

Common gotchas

Forgetting to ack: messages pile up in an "Unacked" state. The queue looks full but consumers aren't processing. Always ack in a try/except and nack with requeue=True on error.

Connection vs channel: one connection per process, one channel per thread. Never share channels across threads.

BlockingConnection in async code: pika's BlockingConnection blocks the event loop. Use aio-pika if you're in async Python.

Nack on failure (with requeue)

def callback(ch, method, properties, body):
    try:
        process(body)
        ch.basic_ack(delivery_tag=method.delivery_tag)
    except Exception as e:
        print(f"Failed: {e}")
        # requeue=False sends to dead-letter queue if configured
        ch.basic_nack(delivery_tag=method.delivery_tag, requeue=True)
