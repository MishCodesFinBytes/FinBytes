Let’s extend our Redis chat demo to simulate multiple workers and show race conditions, TTL expiration issues, and cache misses live. We’ll structure it so you can demonstrate it step by step in a session.

Extended Demo: Multi-Worker Chat + Cache Issues

Goal:

  • Run multiple instances/workers of the chat app to simulate concurrent access.

  • Demonstrate race conditions, cache TTL expiration, cache misses, and Pub/Sub delivery.

  • Use Waitress (Windows-friendly) or Gunicorn (Linux/WSL/Docker).

Step 0: Prerequisites

💡 Tip: Windows → use Waitress; Linux → Gunicorn

Step 1: Single Python File – chat_multi_worker_demo.py

import redis
import threading
import time
from flask import Flask, request, jsonify

# ------------------------------
# Redis Connection
# ------------------------------
r = redis.Redis(host='localhost', port=6379, db=0, decode_responses=True)

# ------------------------------
# Chat + Cache + Logging
# ------------------------------
class ChatDemo:
    MESSAGE_LIST_KEY = "chat_messages"
    CACHE_TTL = 10  # Short TTL to demo expiration quickly
    MAX_MESSAGES = 10  # Keep last N messages

    @staticmethod
    def log(msg):
        print(f"[{time.strftime('%H:%M:%S')}] {msg}")

    @staticmethod
    def start_listener(name="Listener"):
        pubsub = r.pubsub()
        pubsub.subscribe("global_chat")
        ChatDemo.log(f"{name} started listening...")
        for message in pubsub.listen():
            if message['type'] == 'message':
                ChatDemo.log(f"{name} received: {message['data']}")

    @staticmethod
    def send_message(message):
        try:
            # Publish
            r.publish("global_chat", message)
            ChatDemo.log(f"[PUB/SUB SENT] {message}")

            # Cache last N messages atomically
            pipe = r.pipeline()
            pipe.lpush(ChatDemo.MESSAGE_LIST_KEY, message)
            pipe.ltrim(ChatDemo.MESSAGE_LIST_KEY, 0, ChatDemo.MAX_MESSAGES-1)
            pipe.expire(ChatDemo.MESSAGE_LIST_KEY, ChatDemo.CACHE_TTL)
            pipe.execute()
            ChatDemo.log(f"[CACHE] Message cached (TTL={ChatDemo.CACHE_TTL}s)")

        except redis.exceptions.ConnectionError:
            ChatDemo.log("[ERROR] Redis not available!")

    @staticmethod
    def get_cached_messages():
        try:
            messages = r.lrange(ChatDemo.MESSAGE_LIST_KEY, 0, -1)
            if messages:
                ChatDemo.log(f"[CACHE HIT] Retrieved {len(messages)} messages")
            else:
                ChatDemo.log("[CACHE MISS] No messages in cache")
            return messages
        except redis.exceptions.ConnectionError:
            ChatDemo.log("[ERROR] Redis not available!")
            return []

# ------------------------------
# Start two listener threads to simulate multiple workers
# ------------------------------
threading.Thread(target=ChatDemo.start_listener, args=("Listener1",), daemon=True).start()
threading.Thread(target=ChatDemo.start_listener, args=("Listener2",), daemon=True).start()

# ------------------------------
# Flask API
# ------------------------------
app = Flask(__name__)

@app.route("/chat", methods=["POST"])
def send_chat():
    data = request.json
    message = data.get("message")
    if message:
        ChatDemo.send_message(message)
        return jsonify({"status": "sent"})
    return jsonify({"status": "failed", "error": "No message provided"}), 400

@app.route("/history")
def chat_history():
    messages = ChatDemo.get_cached_messages()
    return jsonify({"messages": messages})

@app.route("/ping")
def ping():
    return "Server running!"

# ------------------------------
# Run App
# ------------------------------
if __name__ == "__main__":
    ChatDemo.log("Starting multi-worker chat demo...")
    # Use Flask built-in for quick demo, can replace with Waitress/Gunicorn
    app.run(debug=True)

Step 2: Run Instructions

  1. Start Redis

redis-server.exe

  1. Run Python app

python chat_multi_worker_demo.py

  • Console shows logs for two listener threads (Listener1 and Listener2)

POST /chat {"message":"Hello World!"}

Step 5: Tips for Demo

  1. Adjust CACHE_TTL → 5–15 seconds for fast demo of expiration

  2. Adjust MAX_MESSAGES → simulate eviction of old messages

  3. Stop/start Redis → show connection errors

  4. Add more listener threads → simulate multi-worker concurrency

  5. Check /history → observe cache hits/misses
