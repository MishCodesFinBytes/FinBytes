Measure, don't guess

The first rule of performance optimisation: don't optimise without measurement. Intuition about what's slow is usually wrong. A profiler tells you where time is actually being spent. A micro-benchmark tells you the cost of a specific operation.

Three tools

  • Google Benchmark — micro-benchmarking library. Write a benchmark function, run it, get nanoseconds per operation with statistical analysis. The C++ equivalent of Python's timeit.
  • perf (Linux) — system-level profiler. Shows which functions are hot, cache miss rates, branch mispredictions. No code changes needed.
  • valgrind/callgrind — instruction-level profiler with cache simulation. Slower but detailed. Good for finding cache misses.

Reading assembly

Compiler Explorer (godbolt.org) lets you see the assembly output for any C++ snippet. Essential for verifying that the compiler vectorised your loop, eliminated a copy, or inlined a function. Not as scary as it sounds — you don't need to understand every instruction, just spot the patterns.

Python profiling tools

import timeit
import cProfile
import pstats

# Quick timing
t = timeit.timeit(
    stmt="sum(x**2 for x in range(1000))",
    number=10_000
)
print(f"{t/10_000*1e6:.2f} us per call")

# cProfile — function-level profiling
def price_portfolio(positions: dict) -> float:
    return sum(p * q for p, q in positions.items())

with cProfile.Profile() as pr:
    for _ in range(100_000):
        price_portfolio({"AAPL": 150.0, "MSFT": 300.0})

stats = pstats.Stats(pr)
stats.sort_stats("cumulative")
stats.print_stats(10)

# @profile decorator (injected by kernprof)

@profile
def compute_greeks(S, K, T, r, sigma):
    from scipy.stats import norm
    import numpy as np
    d1 = (np.log(S/K) + (r + 0.5*sigma**2)*T) / (sigma*np.sqrt(T))
    d2 = d1 - sigma * np.sqrt(T)
    delta = norm.cdf(d1)
    gamma = norm.pdf(d1) / (S * sigma * np.sqrt(T))
    return delta, gamma

C# benchmarking with BenchmarkDotNet

// Install: dotnet add package BenchmarkDotNet
using BenchmarkDotNet.Attributes;
using BenchmarkDotNet.Running;

[MemoryDiagnoser]
public class PricingBenchmarks
{
    private double[] _prices = Enumerable.Range(0, 1_000_000)
        .Select(i => 100.0 + i * 0.01).ToArray();

    [Benchmark]
    public double SumLinq() => _prices.Sum();

    [Benchmark]
    public double SumLoop() {
        double sum = 0;
        foreach (var p in _prices) sum += p;
        return sum;
    }

    [Benchmark]
    public double SumSpan() {
        var span = _prices.AsSpan();
        double sum = 0;
        for (int i = 0; i < span.Length; i++) sum += span[i];
        return sum;
    }
}

BenchmarkRunner.Run<PricingBenchmarks>();

Benchmark file

#include <benchmark/benchmark.h>
#include <vector>
#include <numeric>
#include <cmath>

// What we're benchmarking: AoS vs SoA price sum
struct TickAoS { double price; int volume; double time; };

static void BM_AoS(benchmark::State& state) {
    int n = state.range(0);
    std::vector<TickAoS> ticks(n);
    for (int i = 0; i < n; ++i) ticks[i] = {100.0+i*0.01, 1000, i*0.001};

    for (auto _ : state) {
        double sum = 0;
        for (const auto& t : ticks) sum += t.price;
        benchmark::DoNotOptimize(sum);  // prevent optimising away
    }
    state.SetItemsProcessed(state.iterations() * n);
}

static void BM_SoA(benchmark::State& state) {
    int n = state.range(0);
    std::vector<double> prices(n);
    for (int i = 0; i < n; ++i) prices[i] = 100.0 + i*0.01;

    for (auto _ : state) {
        double sum = std::reduce(prices.begin(), prices.end(), 0.0);
        benchmark::DoNotOptimize(sum);
    }
    state.SetItemsProcessed(state.iterations() * n);
}

BENCHMARK(BM_AoS)->Range(1024, 1<<20);
BENCHMARK(BM_SoA)->Range(1024, 1<<20);
BENCHMARK_MAIN();

Compiler Explorer — verifying vectorisation

// Paste this at godbolt.org with g++ -O2 -march=native
// Look for vmovpd / vaddpd / vmulpd — SIMD instructions
#include <vector>

double sum_prices(const double* prices, int n) {
    double sum = 0;
    for (int i = 0; i < n; ++i)
        sum += prices[i];
    return sum;
}
// With -O2: auto-vectorised to process 4 doubles per iteration (AVX2)
// If you see scalar addsd instead of vector vaddpd, vectorisation failed
💡 Always build benchmarks with -O2 or -O3. Debug builds are so different from optimised builds that debug profiling tells you almost nothing useful. And use benchmark::DoNotOptimize() on your result — without it, the compiler may eliminate the entire computation.
