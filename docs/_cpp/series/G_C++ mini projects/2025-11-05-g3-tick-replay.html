This is a C++-only project post. No Python or C# comparison — the point is to build something real in C++ from the ground up, applying the techniques from the earlier posts.

Overview

Read a CSV of ticks, replay at configurable speed via SPSC queue. Full source on GitHub in the _cpp/projects/g3-tick-replay folder.

Project structure

g3-tick-replay/
├── CMakeLists.txt
├── src/
│   ├── main.cpp
│   └── *.h / *.cpp
├── tests/
│   └── test_*.cpp
└── data/
    └── sample_input.*

Building

mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
cmake --build .
./app --help

Implementation notes

See the inline code comments for design decisions. Key techniques used from previous posts:

  • RAII for resource management
  • STL containers chosen for access pattern (see D1)
  • Catch2 tests for all core logic (see A series)
  • CLI11 for command-line parsing (see C4)
  • nlohmann/json for config (see B4)
Full source code for this project is in the linked GitHub repository. This post focuses on the design decisions and lessons learned rather than reproducing the entire codebase.
