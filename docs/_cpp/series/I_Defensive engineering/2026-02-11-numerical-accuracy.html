Silent wrong answers

Crashes announce themselves. Numerical errors do not. A pricer returning 10.4523 when the answer is 10.4521 passes casual inspection but fails desk reconciliation.

Catastrophic cancellation: subtracting two nearly-equal numbers destroys significant digits. Black-Scholes d1-d2 for deep ITM/OTM options, variance E[X^2] - E[X]^2, barrier option payoffs near the barrier - all can suffer this.

NaN and Inf propagation: NaN and Inf propagate silently through arithmetic. A single bad input can corrupt an entire risk report without any exception. Always validate inputs and outputs at system boundaries.

The fix: switching from float to double postpones the problem, it does not solve it. Reformulate the calculation to avoid cancellation, use compensated summation, and validate at boundaries.

// Validate at system boundaries
void validate_inputs(double S, double K, double T, double sigma) {
    auto check = [](double v, const std::string& name) {
        if (!std::isfinite(v)) throw std::invalid_argument(name + " is not finite");
        if (v <= 0) throw std::invalid_argument(name + " must be positive");
    };
    check(S, "S"); check(K, "K"); check(T, "T"); check(sigma, "sigma");
}

// Welford online algorithm - numerically stable variance
double variance_stable(const std::vector<double>& x) {
    double mean = 0, M2 = 0;
    for (std::size_t n = 1; n <= x.size(); ++n) {
        double delta = x[n-1] - mean;
        mean += delta / n;
        M2   += delta * (x[n-1] - mean);
    }
    return M2 / (x.size() - 1);
}

// Kahan compensated sum
double kahan_sum(const std::vector<double>& v) {
    double sum = 0, c = 0;
    for (double xi : v) {
        double y = xi - c, t = sum + y;
        c = (t - sum) - y; sum = t;
    }
    return sum;
}
