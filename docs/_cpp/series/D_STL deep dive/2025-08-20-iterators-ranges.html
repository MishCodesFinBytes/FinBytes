Why iterators exist

Iterators are C++'s abstraction for "a position in a sequence". Every STL algorithm takes iterators — which is why the same std::sort works on vectors, arrays, deques, and custom containers. You write the algorithm once; it works everywhere.

Iterator categories

  • Input — read-once, forward only (network stream)
  • Forward — re-readable, forward only (singly linked list)
  • Bidirectional — forward and backward (std::list, std::map)
  • Random access — jump to any position in O(1) (std::vector, std::array)
  • Contiguous — random access + elements are contiguous in memory (std::vector, std::array, std::span)

C++20 Ranges

Ranges wrap begin/end pairs into a single object. Range adaptors (filter, transform, take, drop) compose with | into lazy pipelines. Similar to Python generators but statically typed and zero-overhead.

Python generators — lazy sequences

from itertools import islice, filterfalse, takewhile, chain

prices = [100.0, 95.0, 102.0, 98.5, 103.0]

# Generators are lazy — values computed on demand
def above_threshold(data, threshold):
    for x in data:
        if x > threshold:
            yield x

gen = above_threshold(prices, 100.0)  # nothing computed yet
first = next(gen)                      # compute first matching element

# itertools pipeline
top3 = list(islice(
    filter(lambda p: p > 95.0, reversed(sorted(prices))),
    3
))

# Generator expression (lazy)
returns = (p / prices[0] - 1 for p in prices)  # not evaluated yet
pos_ret = (r for r in returns if r > 0)

# range() is lazy in Python 3
big_range = range(1_000_000)   # no list created
third     = big_range[2]       # O(1) random access

C# IEnumerable and LINQ

// LINQ is lazy — pipelines don't execute until enumerated
var prices = new[] { 100.0, 95.0, 102.0, 98.5, 103.0 };

// Lazy pipeline — nothing computed here
var pipeline = prices
    .Where(p => p > 100.0)
    .Select(p => p * 1.1)
    .Take(3);

// Executed here
foreach (var p in pipeline) Console.WriteLine(p);

// Custom iterator with yield return
IEnumerable<double> RollingMean(IEnumerable<double> source, int n) {
    var window = new Queue<double>();
    foreach (var x in source) {
        window.Enqueue(x);
        if (window.Count > n) window.Dequeue();
        if (window.Count == n)
            yield return window.Average();
    }
}

var ma5 = RollingMean(prices, 3).ToList();

C++ iterators and C++20 ranges

Custom iterator

#include <iterator>
#include <cstddef>

// Simple forward iterator for a fixed-size ring buffer
template<typename T, std::size_t N>
class RingBuffer {
public:
    void push(T val) {
        buf_[head_] = val;
        head_ = (head_ + 1) % N;
        if (count_ < N) ++count_;
    }

    // Iterator — random access into logical order
    struct iterator {
        using value_type        = T;
        using difference_type   = std::ptrdiff_t;
        using iterator_category = std::forward_iterator_tag;

        const RingBuffer* rb;
        std::size_t       idx;  // logical index

        T operator*() const {
            std::size_t phys = (rb->head_ - rb->count_ + idx + N) % N;
            return rb->buf_[phys];
        }
        iterator& operator++() { ++idx; return *this; }
        bool operator!=(const iterator& o) const { return idx != o.idx; }
    };

    iterator begin() const { return {this, 0}; }
    iterator end()   const { return {this, count_}; }
    std::size_t size() const { return count_; }

private:
    T           buf_[N]{};
    std::size_t head_  = 0;
    std::size_t count_ = 0;
};

C++20 Ranges pipeline

#include <ranges>
#include <vector>
#include <iostream>
#include <numeric>

std::vector<double> prices = {100.0, 95.0, 102.0, 98.5, 103.0, 99.0, 104.5};

namespace rv = std::ranges::views;

// Lazy pipeline: filter above 100, scale by 1.1, take first 3
auto pipeline = prices
    | rv::filter([](double p)    { return p > 100.0; })
    | rv::transform([](double p) { return p * 1.1; })
    | rv::take(3);

for (double p : pipeline)
    std::cout << p << " ";
// Output: 112.2 113.3 114.95

// Rolling mean using iota + sliding window view (C++23)
// C++20 version with manual sliding
auto rolling_mean = [&](int n) {
    std::vector<double> result;
    for (int i = n-1; i < (int)prices.size(); ++i) {
        double sum = std::accumulate(prices.begin()+i-n+1,
                                     prices.begin()+i+1, 0.0);
        result.push_back(sum / n);
    }
    return result;
};

auto ma3 = rolling_mean(3);
💡 C++20 range adaptors are completely lazy and zero-overhead — no temporary vectors, no heap allocations. The compiler sees the entire pipeline at compile time and generates optimal machine code, often equivalent to a hand-written loop.
