Algorithms over loops

The STL ships 100+ algorithms in <algorithm> and <numeric>. They're worth knowing because they're expressive, tested, and often faster than hand-written loops (the compiler can see the intent and optimise accordingly).

The mental model

STL algorithms take iterators (begin/end) and optionally a callable (lambda, function pointer). C++20 Ranges simplifies this: pass the container directly and chain operations with |.

Most useful for finance

  • std::sort / std::partial_sort / std::nth_element
  • std::accumulate / std::reduce (parallel-friendly)
  • std::transform — apply function element-wise
  • std::find_if / std::any_of / std::all_of
  • std::partition / std::stable_partition
  • std::inner_product — dot product

Python functional equivalents

from functools import reduce
import operator

prices = [100.0, 95.0, 102.0, 98.5, 103.0]

# sort
sorted_prices = sorted(prices)           # new list
prices.sort()                            # in-place
top3 = sorted(prices, reverse=True)[:3] # top 3

# map/filter — Python's transform/find_if
returns  = list(map(lambda p: p/prices[0] - 1, prices))
positive = list(filter(lambda r: r > 0, returns))

# reduce — accumulate
total   = reduce(operator.add, prices, 0.0)
product = reduce(operator.mul, prices, 1.0)

# any/all
any_above_100 = any(p > 100 for p in prices)
all_positive  = all(p > 0   for p in prices)

# List comprehensions (preferred over map/filter in Python)
returns_lc  = [p/prices[0] - 1 for p in prices]
positive_lc = [r for r in returns_lc if r > 0]

# zip for inner product
weights    = [0.3, 0.2, 0.2, 0.15, 0.15]
port_value = sum(w * p for w, p in zip(weights, prices))

C# LINQ

var prices = new[] { 100.0, 95.0, 102.0, 98.5, 103.0 };

// Sort
var sorted  = prices.OrderBy(p => p).ToArray();
var top3    = prices.OrderByDescending(p => p).Take(3).ToArray();

// Transform
var returns = prices.Select(p => p / prices[0] - 1).ToArray();

// Filter
var positive = returns.Where(r => r > 0).ToArray();

// Aggregate (reduce)
var total   = prices.Sum();
var product = prices.Aggregate(1.0, (acc, p) => acc * p);

// Any/All
bool anyAbove100 = prices.Any(p => p > 100);
bool allPositive = prices.All(p => p > 0);

// Inner product
var weights    = new[] { 0.3, 0.2, 0.2, 0.15, 0.15 };
var portValue  = prices.Zip(weights, (p, w) => p * w).Sum();

C++ STL algorithms

#include <algorithm>
#include <numeric>
#include <vector>
#include <ranges>   // C++20
#include <iostream>

std::vector<double> prices = {100.0, 95.0, 102.0, 98.5, 103.0};

// Sort
std::sort(prices.begin(), prices.end());                   // ascending in-place
std::sort(prices.begin(), prices.end(), std::greater<>()); // descending
std::nth_element(prices.begin(), prices.begin()+2, prices.end()); // median-partition

// Transform (map equivalent)
std::vector<double> returns(prices.size());
std::transform(prices.begin(), prices.end(), returns.begin(),
    [base = prices[0]](double p) { return p / base - 1.0; });

// Accumulate / reduce
double total   = std::accumulate(prices.begin(), prices.end(), 0.0);
double product = std::accumulate(prices.begin(), prices.end(), 1.0,
    std::multiplies<double>{});

// C++17 parallel reduce
double par_sum = std::reduce(std::execution::par_unseq,
    prices.begin(), prices.end(), 0.0);

// Find
auto it = std::find_if(prices.begin(), prices.end(),
    [](double p) { return p > 100.0; });
if (it != prices.end())
    std::cout << "First above 100: " << *it << "
";

// Any/all
bool any_above = std::any_of(prices.begin(), prices.end(), [](double p) { return p > 100; });
bool all_pos   = std::all_of(prices.begin(), prices.end(), [](double p) { return p > 0;   });

// Partition — split into two groups
auto mid = std::partition(prices.begin(), prices.end(),
    [](double p) { return p > 100.0; });
// [begin, mid) = above 100; [mid, end) = at or below

// Inner product (dot product)
std::vector<double> weights = {0.3, 0.2, 0.2, 0.15, 0.15};
double port_value = std::inner_product(
    prices.begin(), prices.end(), weights.begin(), 0.0);

// C++20 Ranges — pipe syntax
namespace rv = std::ranges::views;
auto above_100 = prices
    | rv::filter([](double p) { return p > 100.0; })
    | rv::transform([](double p) { return p * 1.1; });  // lazy — no allocation

for (double p : above_100)
    std::cout << p << " ";
💡 C++20 Ranges are lazy — the pipeline doesn't execute until you iterate. Chaining filter | transform | take(5) processes only as many elements as needed. This matches Python generator behaviour but with zero overhead.
