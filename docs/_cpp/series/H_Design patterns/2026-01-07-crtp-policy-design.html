Polymorphism without virtual

CRTP (Curiously Recurring Template Pattern) achieves compile-time polymorphism - the flexibility of inheritance without the runtime cost of virtual dispatch. Used extensively in performance-critical C++ libraries such as Eigen, Boost, and STL internals.

A base class template is parameterised by its derived class: template<typename Derived> class Base. The derived class inherits from Base<Derived>. The base can call derived methods without virtual functions, resolved at compile time.

Policy-based design composes behaviour from interchangeable policies - similar to Strategy but at compile time. A pricer parameterised by rate curve policy, vol surface policy, and day count policy, each independently swappable, with zero runtime overhead.

Use CRTP when virtual dispatch overhead is measured to be a problem, when you need static interface enforcement, or when building a framework where users provide their own implementation of hot methods. Otherwise, virtual functions are clearer and adequate.

Python - duck typing handles this

# Python does not need CRTP - duck typing handles compile-time polymorphism
# If it has the right methods, it works

class FlatVolSurface:
    def get_vol(self, strike: float, expiry: float) -> float: return 0.2

class SkewVolSurface:
    def get_vol(self, strike: float, expiry: float) -> float:
        return 0.2 + 0.01 * abs(strike - 100) / 100

class USDAct360:
    def day_fraction(self, days: float) -> float: return days / 360

# Policy-style composition via injection
class Pricer:
    def __init__(self, vol_surface, day_count):
        self._vol = vol_surface
        self._dc  = day_count
    def price(self, S, K, days):
        T   = self._dc.day_fraction(days)
        vol = self._vol.get_vol(K, T)
        return max(S - K, 0) * vol

p1 = Pricer(FlatVolSurface(), USDAct360())
p2 = Pricer(SkewVolSurface(), USDAct360())

C# - generics with constraints

interface IVolSurface { double GetVol(double strike, double expiry); }
interface IDayCount   { double DayFraction(double days); }

class FlatVol : IVolSurface { public double GetVol(double K, double T) => 0.2; }
class SkewVol : IVolSurface { public double GetVol(double K, double T) => 0.2 + 0.01*Math.Abs(K-100)/100; }
class Act360  : IDayCount   { public double DayFraction(double d) => d/360.0; }
class Act365  : IDayCount   { public double DayFraction(double d) => d/365.0; }

class Pricer<TVol, TDc> where TVol : IVolSurface, new()
                          where TDc  : IDayCount,   new()
{
    private readonly TVol _vol = new();
    private readonly TDc  _dc  = new();
    public double Price(double S, double K, double days) {
        double T   = _dc.DayFraction(days);
        double vol = _vol.GetVol(K, T);
        return Math.Max(S - K, 0) * vol;
    }
}

var p1 = new Pricer<FlatVol, Act360>();
var p2 = new Pricer<SkewVol, Act365>();

C++ CRTP and policy-based design

// CRTP: base calls derived methods without virtual
template<typename Derived>
class PricerBase {
public:
    double price(double S, double K, double T) const {
        return static_cast<const Derived*>(this)->price_impl(S, K, T);
    }
    double pv01(double S, double K, double T, double notional) const {
        return price(S, K, T) * notional * 0.0001;
    }
};

struct BlackScholesPricer : PricerBase<BlackScholesPricer> {
    double price_impl(double S, double K, double T) const {
        return std::max(S - K, 0.0) * 1.05;
    }
};

struct IntrinsicPricer : PricerBase<IntrinsicPricer> {
    double price_impl(double S, double K, double T) const {
        return std::max(S - K, 0.0);
    }
};

BlackScholesPricer bs;
double px = bs.price(100, 100, 1);    // no virtual call
double pv = bs.pv01(100, 100, 1, 1'000'000);

// Policy-based design
struct FlatVol { double get_vol(double K, double T) const { return 0.2; } };
struct SkewVol { double get_vol(double K, double T) const { return 0.2 + 0.01*std::abs(K-100)/100; } };
struct Act360  { double day_fraction(double d) const { return d/360.0; } };
struct Act365  { double day_fraction(double d) const { return d/365.0; } };

template<typename VolPolicy, typename DayCountPolicy>
class PolicyPricer : private VolPolicy, private DayCountPolicy {
public:
    double price(double S, double K, double days) const {
        double T   = DayCountPolicy::day_fraction(days);
        double vol = VolPolicy::get_vol(K, T);
        return std::max(S - K, 0.0) * vol;
    }
};

PolicyPricer<FlatVol, Act360> p1;  // zero overhead - policies inlined
PolicyPricer<SkewVol, Act365> p2;
CRTP is an advanced pattern. Measure whether virtual dispatch is your bottleneck before using it. For most pricing code, virtual functions are adequate. CRTP shines when you call the same virtual method millions of times in a tight loop, or when building a framework where users plug in their own hot implementations.
