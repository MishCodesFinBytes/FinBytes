When the consumer cannot keep up

A market data feed can publish 1 million ticks per second. A pricer might handle 50,000 per second. Without backpressure, the queue grows without bound until the process is killed by OOM.

Three strategies:

  • Block: the producer waits when the queue is full. Correct but introduces latency into the producer - risky if the producer is time-sensitive.
  • Drop: discard new items when full. Lose data but the producer is never blocked. Appropriate when old data is less valuable than new (market data: drop old prices, keep latest).
  • Shed: reject work at the API level before it enters the queue. Caller receives an error and can retry. Appropriate for RPC-style requests where the caller can back off.

Key insight: backpressure is a signal, not an error. A queue reaching capacity is telling you the consumer is too slow. Monitor queue depth continuously - alert at 70%, page at 85%.

Python backpressure

from queue import Queue, Full
import threading

# Drop strategy: keep only the latest item per symbol
class LatestOnlyQueue:
    def __init__(self): self._data: dict = {}; self._lock = threading.Lock()
    def put(self, symbol: str, price: float) -> None:
        with self._lock: self._data[symbol] = price  # overwrites old price
    def get_all(self) -> dict:
        with self._lock: items = dict(self._data); self._data.clear(); return items

feed_queue = Queue(maxsize=1000)

# Rate limiting at API level (shed)
import time
class RateLimitedPricer:
    def __init__(self, max_rps: int):
        self._max_rps = max_rps; self._window = []; self._lock = threading.Lock()
    def price(self, trade):
        now = time.monotonic()
        with self._lock:
            self._window = [t for t in self._window if now - t < 1.0]
            if len(self._window) >= self._max_rps:
                raise RuntimeError("Rate limit exceeded - retry later")
            self._window.append(now)
        return self._price_impl(trade)

C++ latest-value cache and drop counting

#include <atomic>
#include <array>
#include <optional>

// Latest-value cache: producer overwrites, consumer reads latest
// No queue needed for market data where only the latest price matters
template<typename T, std::size_t N>
class LatestValueCache {
    struct Slot { std::atomic<uint64_t> seq{0}; T value{}; };
    std::array<Slot, N> slots_;
public:
    void put(std::size_t idx, T val) {
        auto& s = slots_[idx % N];
        s.seq.fetch_add(1, std::memory_order_release);
        s.value = val;
        s.seq.fetch_add(1, std::memory_order_release);
    }
    std::optional<T> get(std::size_t idx) const {
        auto& s = slots_[idx % N];
        uint64_t seq1 = s.seq.load(std::memory_order_acquire);
        if (seq1 & 1) return std::nullopt;   // write in progress
        T val = s.value;
        uint64_t seq2 = s.seq.load(std::memory_order_acquire);
        if (seq1 != seq2) return std::nullopt;  // concurrent write
        return val;
    }
};

// Non-blocking push with drop counter
std::atomic<uint64_t> dropped_ticks{0};

// In producer:
if (!queue.push(tick, std::chrono::milliseconds(0))) {
    dropped_ticks.fetch_add(1, std::memory_order_relaxed);
}
