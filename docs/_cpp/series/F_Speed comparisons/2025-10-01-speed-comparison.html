Raw numbers

Benchmarks are lies — they depend on hardware, compiler, version, and what you're measuring. These are order-of-magnitude comparisons on a modern x86 laptop (Ryzen 7, GCC 13 / CPython 3.12 / .NET 8), intended to build intuition, not to be cited.

The headline

For CPU-bound numerical work: C++ is 10–100x faster than Python. C# is typically 5–20x faster than Python and 2–5x slower than C++. Python with NumPy narrows the gap significantly for vectorisable operations — but only those.

Where Python wins

Development speed. Iteration time. Library ecosystem (pandas, scipy, sklearn, plotly). For prototyping, data wrangling, and research, Python's productivity advantage far outweighs the runtime cost.

Where C++ wins

Tight numerical loops, latency-sensitive paths, memory-constrained environments, and code that needs to be fast and correct at the same time as doing I/O. Also: the only language here with predictable latency — no GC pauses.

Python benchmarks

import timeit
import numpy as np

N = 1_000_000

# Integer sum
t = timeit.timeit(lambda: sum(range(N)), number=10)
print(f"int sum (Python loop): {t/10*1000:.1f}ms")

# Float sum with NumPy
arr = np.random.random(N)
t = timeit.timeit(lambda: arr.sum(), number=1000)
print(f"float sum (NumPy):     {t/1000*1000:.3f}ms")

# Pure Python float loop
prices = [float(i) for i in range(N)]
t = timeit.timeit(lambda: sum(prices), number=10)
print(f"float sum (py loop):   {t/10*1000:.1f}ms")

# Dict lookup
d = {str(i): i for i in range(10_000)}
keys = [str(i) for i in range(10_000)]
t = timeit.timeit(lambda: [d[k] for k in keys], number=1000)
print(f"dict lookup (10k):     {t/1000*1000:.3f}ms")

# String concatenation
parts = ["word"] * 1000
t = timeit.timeit(lambda: "".join(parts), number=10_000)
print(f"string join (1k):      {t/10_000*1e6:.1f}us")

# Typical results:
# int sum (Python loop): ~45ms    (C++: ~0.5ms)
# float sum (NumPy):     ~0.4ms   (C++: ~0.3ms — nearly equal!)
# float sum (py loop):   ~50ms    (C++: ~0.5ms)
# dict lookup (10k):     ~0.4ms   (C++: ~0.1ms)
# string join (1k):      ~3us     (C++: ~1us)
NumPy's float sum is nearly as fast as C++ because it calls the same SIMD-vectorised BLAS routines under the hood. The Python overhead is amortised over the entire array.

C# benchmarks (BenchmarkDotNet)

using BenchmarkDotNet.Attributes;
using BenchmarkDotNet.Running;

[MemoryDiagnoser]
public class Benchmarks {
    const int N = 1_000_000;
    private readonly double[] _arr = Enumerable.Range(0,N).Select(i=>(double)i).ToArray();
    private readonly Dictionary<string,int> _dict;

    public Benchmarks() {
        _dict = Enumerable.Range(0,10_000).ToDictionary(i=>i.ToString(), i=>i);
    }

    [Benchmark] public long IntSum() {
        long s = 0; for(int i=0;i<N;i++) s+=i; return s;
    }
    [Benchmark] public double FloatSum() {
        double s = 0; foreach(var x in _arr) s+=x; return s;
    }
    [Benchmark] public double FloatSumLinq() => _arr.Sum();
    [Benchmark] public int DictLookup() {
        int s = 0;
        for(int i=0;i<10_000;i++) s+=_dict[i.ToString()];
        return s;
    }
}

// Typical results:
// IntSum:       ~0.5ms   (C++: ~0.5ms — same)
// FloatSum:     ~0.5ms   (C++: ~0.5ms — same for simple loops)
// FloatSumLinq: ~1.2ms   (overhead of LINQ machinery)
// DictLookup:   ~0.4ms   (C++: ~0.15ms)

C++ benchmarks (Google Benchmark)

#include <benchmark/benchmark.h>
#include <numeric>
#include <vector>
#include <unordered_map>
#include <string>
#include <random>

constexpr int N = 1'000'000;

static void BM_IntSum(benchmark::State& state) {
    for (auto _ : state) {
        long long s = 0;
        for (int i = 0; i < N; ++i) s += i;
        benchmark::DoNotOptimize(s);
    }
}
BENCHMARK(BM_IntSum);  // ~0.5ms

static void BM_FloatSum(benchmark::State& state) {
    std::vector<double> v(N);
    std::iota(v.begin(), v.end(), 0.0);
    for (auto _ : state) {
        double s = std::reduce(v.begin(), v.end(), 0.0);
        benchmark::DoNotOptimize(s);
    }
}
BENCHMARK(BM_FloatSum);  // ~0.3ms (auto-vectorised with AVX2)

static void BM_DictLookup(benchmark::State& state) {
    std::unordered_map<std::string,int> m;
    m.reserve(10'000);
    for (int i = 0; i < 10'000; ++i) m[std::to_string(i)] = i;

    for (auto _ : state) {
        int s = 0;
        for (int i = 0; i < 10'000; ++i) s += m[std::to_string(i)];
        benchmark::DoNotOptimize(s);
    }
}
BENCHMARK(BM_DictLookup);  // ~0.15ms

BENCHMARK_MAIN();
💡 The C++ float sum is faster than Python by ~100x (pure loop) but only ~1.3x faster than NumPy — because NumPy uses the same AVX2 instructions. The lesson: Python with NumPy is nearly as fast as C++ for vectorisable array operations. The gap is in loop overhead, object creation, and non-vectorisable logic.
