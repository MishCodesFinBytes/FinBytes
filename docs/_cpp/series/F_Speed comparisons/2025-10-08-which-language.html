The honest answer: it depends

There's no universally correct language. The question is: what are your constraints? Latency? Development speed? Team familiarity? Library ecosystem? Long-term maintainability? Here's how I think about it after years in all three.

Decision matrix

  • Research / prototyping → Python. Fast iteration, Jupyter, pandas, scipy.
  • Data wrangling (CSV, Excel, APIs) → Python or C#. C++ is painful here.
  • Pricing library (batch) → C++ or C#. Python with pybind11 if you need Python interface.
  • Real-time / low-latency → C++ only. No GC pauses, predictable latency.
  • Risk engine (EOD batch) → C# or C++. C# wins on developer productivity.
  • CLI tools and scripts → Python for small/ad-hoc, C++ for performance-critical.
  • GUI/web front-end → C# (WPF/Blazor) or Python (Dash/Streamlit). Not C++.

The right architecture

Most serious quant systems use all three: Python for research and notebooks, C# for middle-office tooling and reporting, C++ for the execution-critical path. pybind11 and COM interop bridge them.

def calibrate_vol(market_price, S, K, T, r):
    def objective(sigma):
        d1 = (np.log(S/K) + (r + 0.5*sigma**2)*T) / (sigma*np.sqrt(T))
        d2 = d1 - sigma*np.sqrt(T)
        model = S*norm.cdf(d1) - K*np.exp(-r*T)*norm.cdf(d2)
        return (model - market_price)**2
    return minimize(objective, x0=0.2, method="L-BFGS-B",
                    bounds=[(0.01, 2.0)]).x[0]

C++ — where it shines

// 1. Real-time market data handler — microsecond latency
class MarketDataHandler {
    SPSCQueue<Tick, 65536> queue_;

public:
    void on_tick(const Tick& t) {  // called at 1M+ ticks/sec
        queue_.push(t);           // lock-free, ~50ns
    }
};

// 2. Intraday risk engine — no GC pauses
class RiskEngine {
    // Pre-allocated pools — no allocation in hot path
    std::array<GreekResult, MAX_POSITIONS> results_{};

public:
    void reprice_all(std::span<const Position> positions) {
        // Tight loop — auto-vectorised, cache-friendly SoA layout
        for (std::size_t i = 0; i < positions.size(); ++i)
            results_[i] = pricer_.price(positions[i]);
    }
};

// 3. Order book — microsecond latency requirement
class OrderBook {
    std::map<Price, Level, std::greater<Price>> bids_;
    std::map<Price, Level>                       asks_;

public:
    // Add order: O(log n) — typically <500ns
    void add(const Order& o);

    // Best bid/ask: O(1)
    Price best_bid() const { return bids_.begin()->first; }
    Price best_ask() const { return asks_.begin()->first; }
};
💡 The right mental model: Python is for thinking, C# is for building, C++ is for optimising. Start in Python, move performance-critical components to C++ via pybind11 when you have measured evidence that it matters.
