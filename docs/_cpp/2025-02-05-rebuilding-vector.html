Why build what already exists?

Understanding how std::vector works internally changes how you use it. Once you've built one yourself — the capacity doubling, the placement new, the iterator invalidation rules — none of its behaviour surprises you again.

What std::vector actually is

A dynamic array: a contiguous block of memory on the heap, a size (elements currently stored), and a capacity (elements the block can hold before reallocation). When size reaches capacity, it allocates a larger block, moves everything across, and frees the old one.

Capacity doubling

Why double? Amortised O(1) push_back. If you grow by 1 each time, every push_back copies everything — O(n) per operation. If you double capacity when full, each element is moved at most log₂(n) times across its lifetime — O(1) amortised.

Placement new

Normal new allocates memory and constructs an object. When building a vector, you allocate a raw memory block upfront, then construct objects into it as elements are added. That requires placement new — construct at a specific address without allocating.

Iterator invalidation

Any operation that causes reallocation (push_back beyond capacity, insert, reserve) invalidates all pointers, references, and iterators into the vector. This is why storing vec[0]& across a push_back is undefined behaviour.

Python list — similar concept, different implementation

Python's list is also a dynamic array under the hood, but stores pointers to heap objects rather than the objects themselves. Growth factor is ~1.125 rather than 2.

import sys

lst = []
prev_size = sys.getsizeof(lst)
for i in range(20):
    lst.append(i)
    size = sys.getsizeof(lst)
    if size != prev_size:
        print(f"len={len(lst)}: reallocated to {size} bytes")
        prev_size = size

# Output shows reallocation points as capacity is exceeded

Building a simple dynamic array in Python

class DynArray:
    def __init__(self):
        self._data = [None] * 1
        self._size = 0
        self._cap  = 1

    def append(self, item):
        if self._size == self._cap:
            self._grow()
        self._data[self._size] = item
        self._size += 1

    def _grow(self):
        new_cap  = self._cap * 2
        new_data = [None] * new_cap
        for i in range(self._size):
            new_data[i] = self._data[i]
        self._data = new_data
        self._cap  = new_cap

    def __getitem__(self, i): return self._data[i]
    def __len__(self):        return self._size

arr = DynArray()
for i in range(10):
    arr.append(i)
print([arr[i] for i in range(len(arr))])

C# List<T> — same idea, managed memory

List<T> in C# is also a dynamic array, with a default initial capacity of 4 and doubling growth. The GC handles memory — no placement new, no manual cleanup.

using System.Reflection;

var list = new List<double>();
int prevCap = list.Capacity;

for (int i = 0; i < 20; i++) {
    list.Add(i);
    if (list.Capacity != prevCap) {
        Console.WriteLine($"Count={list.Count}: capacity -> {list.Capacity}");
        prevCap = list.Capacity;
    }
}
// Count=1:  capacity -> 4
// Count=5:  capacity -> 8
// Count=9:  capacity -> 16
// Count=17: capacity -> 32

Pre-allocating capacity

// If you know the size upfront, reserve it
var prices = new List<double>(capacity: 100_000);
// No reallocations as long as count stays under 100,000

C++ — building Vec<T> from scratch

#include <cstddef>
#include <stdexcept>
#include <utility>
#include <new>
#include <iostream>

template<typename T>
class Vec {
public:
    Vec() = default;

    explicit Vec(std::size_t n, const T& val = T{}) {
        reserve(n);
        for (std::size_t i = 0; i < n; ++i)
            push_back(val);
    }

    ~Vec() {
        clear();
        ::operator delete(data_);
    }

    // Disable copy for simplicity (real vector supports it)
    Vec(const Vec&)            = delete;
    Vec& operator=(const Vec&) = delete;

    // Move constructor
    Vec(Vec&& other) noexcept
        : data_(other.data_), size_(other.size_), cap_(other.cap_)
    {
        other.data_ = nullptr;
        other.size_ = other.cap_ = 0;
    }

    void push_back(const T& val) {
        if (size_ == cap_) grow();
        new(data_ + size_) T(val);  // placement new
        ++size_;
    }

    void push_back(T&& val) {
        if (size_ == cap_) grow();
        new(data_ + size_) T(std::move(val));  // placement new (move)
        ++size_;
    }

    void reserve(std::size_t new_cap) {
        if (new_cap <= cap_) return;
        reallocate(new_cap);
    }

    T&       operator[](std::size_t i)       { return data_[i]; }
    const T& operator[](std::size_t i) const { return data_[i]; }

    T*       begin()       { return data_; }
    T*       end()         { return data_ + size_; }
    const T* begin() const { return data_; }
    const T* end()   const { return data_ + size_; }

    std::size_t size()     const { return size_; }
    std::size_t capacity() const { return cap_; }
    bool        empty()    const { return size_ == 0; }

    void clear() {
        for (std::size_t i = 0; i < size_; ++i)
            data_[i].~T();  // explicitly call destructor
        size_ = 0;
    }

private:
    void grow() {
        reallocate(cap_ == 0 ? 1 : cap_ * 2);
    }

    void reallocate(std::size_t new_cap) {
        // Allocate raw memory — no construction yet
        T* new_data = static_cast<T*>(::operator new(new_cap * sizeof(T)));

        // Move existing elements into new buffer
        for (std::size_t i = 0; i < size_; ++i) {
            new(new_data + i) T(std::move(data_[i]));
            data_[i].~T();
        }

        ::operator delete(data_);
        data_ = new_data;
        cap_  = new_cap;
    }

    T*          data_ = nullptr;
    std::size_t size_ = 0;
    std::size_t cap_  = 0;
};

int main() {
    Vec<double> v;
    for (int i = 0; i < 10; ++i) {
        v.push_back(i * 1.1);
        std::cout << "size=" << v.size()
                  << " cap=" << v.capacity() << "\n";
    }

    double sum = 0;
    for (auto x : v) sum += x;
    std::cout << "sum=" << sum << "\n";
    return 0;
}
💡 ::operator new(n) allocates n bytes of raw memory without constructing anything. new(ptr) T(args) (placement new) constructs a T at address ptr without allocating. Together they let you separate allocation from construction — the core of how containers work.

Iterator invalidation in practice

#include <vector>
#include <iostream>

int main() {
    std::vector<double> v = {1.0, 2.0, 3.0};

    double& ref = v[0];          // reference to first element
    auto*   ptr = v.data();      // pointer to buffer

    v.push_back(4.0);            // may reallocate!

    // ref and ptr are now dangling — undefined behaviour
    // std::cout << ref << "\n";  // DO NOT DO THIS

    // Safe: re-index after any mutating operation
    std::cout << v[0] << "\n";   // fine — v[0] is always valid
    return 0;
}
