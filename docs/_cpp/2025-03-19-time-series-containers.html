The problem with naive tick storage

A naive implementation stores ticks in a growing vector, allocates on every push, and copies the whole thing for rolling windows. For real-time systems processing millions of ticks per second, this doesn't work.

What we need

  • Fixed-capacity circular buffer — O(1) push, O(1) random access, no allocation after construction
  • SoA layout — separate arrays for price, volume, time for cache-friendly loop performance
  • Pre-allocated — all memory reserved upfront, no dynamic allocation in the hot path

Circular buffer

A fixed-size array with a head pointer. New elements overwrite the oldest. Capacity is fixed — no reallocation. Perfect for rolling windows: always keep the last N ticks without any shifting or copying.

Python tick storage

from collections import deque
from dataclasses import dataclass
import time

@dataclass
class OHLCV:
    open:   float
    high:   float
    low:    float
    close:  float
    volume: int

# deque as circular buffer
class TickBuffer:
    def __init__(self, capacity: int):
        self.buf = deque(maxlen=capacity)

    def push(self, bar: OHLCV) -> None:
        self.buf.append(bar)

    def close_prices(self) -> list[float]:
        return [b.close for b in self.buf]

    def __len__(self) -> int:
        return len(self.buf)

buf = TickBuffer(256)
for i in range(300):
    buf.push(OHLCV(100+i, 102+i, 99+i, 101+i, 1000))
print(len(buf))            # 256 — older bars dropped
print(buf.close_prices()[-1])  # 400.0
Python's deque(maxlen=N) is a circular buffer — O(1) append, O(1) pop, auto-drops oldest when full. Internally it's a doubly-linked list of fixed-size blocks, not a single contiguous array — less cache-friendly than C++.

C# circular buffer

struct OHLCV {
    public double Open, High, Low, Close;
    public int    Volume;
}

class CircularBuffer<T> {
    private readonly T[] _buf;
    private int _head, _count;

    public CircularBuffer(int capacity) {
        _buf = new T[capacity];
    }

    public void Push(T item) {
        _buf[_head] = item;
        _head = (_head + 1) % _buf.Length;
        if (_count < _buf.Length) _count++;
    }

    public T this[int i] => _buf[(_head - _count + i + _buf.Length) % _buf.Length];
    public int Count => _count;
}

var bars = new CircularBuffer<OHLCV>(256);
for (int i = 0; i < 300; i++)
    bars.Push(new OHLCV { Close = 100 + i });
Console.WriteLine(bars.Count);  // 256

C++ circular tick buffer — SoA layout

#include <array>
#include <cstddef>
#include <stdexcept>
#include <numeric>
#include <iostream>

// Compile-time capacity — all on the stack if small enough
template<std::size_t N>
class OHLCVBuffer {
public:
    struct Bar {
        double open, high, low, close;
        int    volume;
    };

    void push(const Bar& bar) {
        open_[head_]   = bar.open;
        high_[head_]   = bar.high;
        low_[head_]    = bar.low;
        close_[head_]  = bar.close;
        volume_[head_] = bar.volume;
        head_ = (head_ + 1) % N;
        if (count_ < N) ++count_;
    }

    // Access by logical index (0 = oldest)
    double close(std::size_t i) const {
        return close_[logical_to_physical(i)];
    }

    // Mean of close prices — tight loop over contiguous doubles
    double mean_close() const {
        if (count_ == 0) return 0.0;
        double sum = 0;
        for (std::size_t i = 0; i < count_; ++i)
            sum += close_[i];  // raw array — fully vectorisable
        return sum / count_;
    }

    std::size_t size()     const { return count_; }
    bool        full()     const { return count_ == N; }

private:
    std::size_t logical_to_physical(std::size_t i) const {
        return (head_ - count_ + i + N) % N;
    }

    // SoA — each field in its own contiguous array
    std::array<double, N> open_{};
    std::array<double, N> high_{};
    std::array<double, N> low_{};
    std::array<double, N> close_{};
    std::array<int, N>    volume_{};
    std::size_t head_  = 0;
    std::size_t count_ = 0;
};

int main() {
    OHLCVBuffer<256> buf;  // 256 bars, stack allocated

    for (int i = 0; i < 300; ++i)
        buf.push({100.0+i, 102.0+i, 99.0+i, 101.0+i, 1000});

    std::cout << "size:  "       << buf.size()       << "\n";  // 256
    std::cout << "mean close: " << buf.mean_close()  << "\n";
    return 0;
}
💡 With N as a template parameter, the buffer size is known at compile time. std::array lives on the stack for small N — zero heap allocation. The SoA layout means mean_close() touches only the close_ array — perfect cache behaviour.
