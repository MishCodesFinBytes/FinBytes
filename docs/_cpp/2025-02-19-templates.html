Write once, compile for any type

Templates are C++'s mechanism for generic programming — write code once, have the compiler generate type-specific versions automatically. std::vector<double> and std::vector<int> are both generated from the same template.

Compile-time, not runtime

This is the key difference from Python generics (type hints, runtime duck typing) and C# generics (runtime type parameter, JIT specialisation). C++ templates are resolved entirely at compile time — the compiler generates separate machine code for each type you use. Zero runtime overhead, but longer compile times.

Function templates

Write a function that works for any type satisfying certain operations. The compiler instantiates a version for each type it's called with.

Class templates

Write a class parameterised by one or more types. std::vector, std::map, std::unique_ptr are all class templates.

Template specialisation

Provide a custom implementation for a specific type. Useful for optimising hot paths — e.g. a special version of your pricer for float vs double.

Concepts (C++20)

Named constraints on template parameters. Instead of a cryptic error when the wrong type is passed, you get a clear message at the call site. The modern replacement for SFINAE.

Python — duck typing and type hints

Python is dynamically typed — any function works with any type that has the right methods. Type hints are documentation for the type checker, not enforced at runtime.

from typing import TypeVar, Generic

# Works for any type with + and / operators
def mean(values):
    return sum(values) / len(values)

print(mean([1.0, 2.0, 3.0]))  # 2.0
print(mean([1, 2, 3]))         # 2.0 — works for int too

# Type hints for documentation
T = TypeVar('T', int, float)

def mean_typed(values: list[T]) -> float:
    return sum(values) / len(values)

Generic class with TypeVar

from typing import Generic, TypeVar

T = TypeVar('T')

class Stack(Generic[T]):
    def __init__(self) -> None:
        self._items: list[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

    def peek(self) -> T:
        return self._items[-1]

s: Stack[float] = Stack()
s.push(1.0)
s.push(2.0)
print(s.pop())  # 2.0
Python generics are purely a type-checking tool. The runtime doesn't specialise — a Stack[float] and Stack[int] are the same class at runtime. C++ generates completely separate machine code for each instantiation.

C# generics — runtime type parameters

C# generics are reified — the type parameter exists at runtime. List<double> and List<int> are distinct types at runtime, but value types are specialised by the JIT while reference types share one compiled form.

// Generic function
static T Max<T>(T a, T b) where T : IComparable<T>
    => a.CompareTo(b) >= 0 ? a : b;

Console.WriteLine(Max(1.0, 2.0));   // 2.0
Console.WriteLine(Max("a", "b"));   // b

// Generic class
public class Stack<T> {
    private readonly List<T> _items = new();
    public void Push(T item) => _items.Add(item);
    public T    Pop()        => _items[^1]; // also removes
    public T    Peek()       => _items[^1];
    public int  Count        => _items.Count;
}

var s = new Stack<double>();
s.Push(1.0);
s.Push(2.0);
C# constraints (where T : IComparable<T>) are like C++20 concepts — they restrict what types can be used and produce readable errors at the call site.

C++ templates

Function template

#include <iostream>
#include <vector>
#include <numeric>

// Works for any numeric type
template<typename T>
T mean(const std::vector<T>& values) {
    if (values.empty()) return T{};
    return std::accumulate(values.begin(), values.end(), T{}) / values.size();
}

int main() {
    std::vector<double> d = {1.0, 2.0, 3.0};
    std::vector<float>  f = {1.0f, 2.0f, 3.0f};

    std::cout << mean(d) << "\n";  // compiler instantiates mean<double>
    std::cout << mean(f) << "\n";  // compiler instantiates mean<float>
    return 0;
}

Class template

#include <stdexcept>
#include <vector>

template<typename T>
class Stack {
public:
    void push(const T& val) { data_.push_back(val); }
    void push(T&& val)      { data_.push_back(std::move(val)); }

    T pop() {
        if (empty()) throw std::underflow_error("empty stack");
        T val = std::move(data_.back());
        data_.pop_back();
        return val;
    }

    const T& peek() const {
        if (empty()) throw std::underflow_error("empty stack");
        return data_.back();
    }

    bool        empty() const { return data_.empty(); }
    std::size_t size()  const { return data_.size();  }

private:
    std::vector<T> data_;
};

// Usage
Stack<double> prices;
prices.push(100.5);
prices.push(101.0);
std::cout << prices.pop() << "\n";  // 101.0

Template specialisation

#include <cmath>

// General template
template<typename T>
T square_root(T x) {
    return static_cast<T>(std::sqrt(static_cast<double>(x)));
}

// Full specialisation for float — use sqrtf directly
template<>
float square_root<float>(float x) {
    return std::sqrtf(x);
}

// Now square_root<float> calls sqrtf, square_root<double> calls sqrt

C++20 Concepts — readable constraints

#include <concepts>
#include <vector>
#include <numeric>

// Constraint: T must support arithmetic operations
template<typename T>
concept Numeric = std::is_arithmetic_v<T>;

template<Numeric T>
T mean(const std::vector<T>& v) {
    return std::accumulate(v.begin(), v.end(), T{}) / v.size();
}

// mean<std::string> now gives a clear error at the call site:
// "constraint 'Numeric' not satisfied"
// Instead of a wall of template instantiation errors
💡 Before C++20, you'd use SFINAE (std::enable_if) to constrain templates — cryptic and verbose. Concepts are the modern, readable replacement. Use them in all new code targeting C++20 or later.

Non-type template parameters — compile-time constants

#include <array>
#include <numeric>

// N is a compile-time constant — no dynamic allocation
template<typename T, std::size_t N>
class RollingWindow {
public:
    void push(T val) {
        buf_[head_] = val;
        head_ = (head_ + 1) % N;
        if (count_ < N) ++count_;
    }

    T mean() const {
        T sum = std::accumulate(buf_.begin(), buf_.begin() + count_, T{});
        return sum / count_;
    }

    std::size_t size() const { return count_; }

private:
    std::array<T, N> buf_{};
    std::size_t head_  = 0;
    std::size_t count_ = 0;
};

// Instantiate a 20-period rolling window — all on the stack
RollingWindow<double, 20> ma20;
for (double p : {100.0, 101.0, 99.5, 102.0})
    ma20.push(p);
