Mastering Python Patch Targets: Top-Level Functions, Instance Methods, and Static Methods

Python’s unittest.mock.patch is a powerful tool for testing. But when multiple functions or methods share the same name—across classes, static methods, or even top-level functions—developers often struggle with patch targets.

This post will show exactly how to patch each type of function/method, explain why each target works, and provide fully runnable examples.

By the end, you’ll never wonder whether to patch helpers.func or service.Class.func again.

1️⃣ Setup: Two Classes + Top-Level Function + Static Method

service.py# Top-level function def compute(x): return x * 2 # First class class CalculatorA: def compute(self, x): return x + 1 @staticmethod def compute_static(x): return x * 10 @classmethod def compute_class(cls, x): return x * 100 # Second class class CalculatorB: def compute(self, x): return x + 10

2️⃣ Patch Examples: pytest Tests

test_service.pyimport pytest from unittest.mock import patch from service import compute, CalculatorA, CalculatorB # Patch top-level function @patch("service.compute") def test_patch_top(mock_compute): mock_compute.return_value = 99 assert compute(5) == 99 # Class methods remain unaffected assert CalculatorA().compute(5) == 6 assert CalculatorB().compute(5) == 15 # Patch CalculatorA instance method @patch("service.CalculatorA.compute") def test_patch_calc_a(mock_method): mock_method.return_value = 100 assert CalculatorA().compute(5) == 100 # Other methods unaffected assert CalculatorB().compute(5) == 15 assert compute(5) == 10 # Patch CalculatorB instance method @patch("service.CalculatorB.compute") def test_patch_calc_b(mock_method): mock_method.return_value = 200 assert CalculatorB().compute(5) == 200 # Other methods unaffected assert CalculatorA().compute(5) == 6 assert compute(5) == 10 # Patch CalculatorA static method @patch("service.CalculatorA.compute_static") def test_patch_static(mock_static): mock_static.return_value = 999 assert CalculatorA.compute_static(5) == 999 # Instance method and top-level function unaffected assert CalculatorA().compute(5) == 6 assert compute(5) == 10 # Patch CalculatorA class method @patch("service.CalculatorA.compute_class") def test_patch_class(mock_class): mock_class.return_value = 555 assert CalculatorA.compute_class(5) == 555 # Nothing else affected assert CalculatorA().compute(5) == 6 assert compute(5) == 10

4️⃣ Why This Works

Python resolves names at runtime:

Top-level functions → module namespace Instance methods → class namespace Static methods / Class methods → class namespace Multiple classes with same method name → patch only affects the class specified

Rule of thumb:

Always patch where Python looks up the object, not where it was defined.

5️⃣ Bonus: Patch in Actionprint(compute(5)) # 10 print(CalculatorA().compute(5)) # 6 print(CalculatorB().compute(5)) # 15 print(CalculatorA.compute_static(5)) # 50 print(CalculatorA.compute_class(5)) # 500

After patching each target, only the patched object changes.

✅ Key Takeaways

Top-level function: patch "module.function_name" Instance method: patch "module.ClassName.method_name" Static method: patch "module.ClassName.method_name" Class method: patch "module.ClassName.method_name" Multiple classes or overlapping names: patch is scoped per class, does not interfere with others
